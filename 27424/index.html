NPM

发表于 2020-04-18 更新于 2021-10-07 分类于 npm 阅读次数： 本文字数： 7.2k 阅读时长 ≈ 12 分钟

NPM 是 NodeJS 下的包（Module 模块）管理工具。在 NodeJS v0.6.x 之后的版本中，嵌套于 NodeJS 系统中，随 NodeJS 一起发布。通过 NPM 可以协助开发者安装、卸载、删除、更新 NodeJS 套件，并且可以通过 NPM 发布自己的插件。

安装 NPM

  1. 通过 NodeJS 官网直接下载 LTS 稳定版本的 PKG 包安装
  2. 通过 NVM 管理 NodeJS 安装，适用于复杂的项目

NVM 安装 NodeJS 之前需要先安装 HomeBrew

通过 HomeBrew 安装的 NVM，缺少 nvm.sh 执行脚本文件，还需要配置 ~/.bash_profile、~/.bashrc 或 ~/.zshrc 开发环境

配置完成后，source 命令重启下相应的环境配置文件

查看 NPM 版本

更新 NPM 版本

NPM 版本要比 NodeJS 的更新的要快，有时我们安装的不一定是最新版本，大家可通过下述命令单独更新 NPM

由于国内政策可能存在网速过慢问题，推荐大家使用淘宝的CNPM进行解决

更改 NPM 安装目录

NPM 默认安装在 /usr/local 目录下，通过命令可获悉安装位置

如果我们想更改它的安装位置，可以使用 npm config set prefix 命令对其修改

位置更改完成后，我们需要重新安装 NPM，并且需要更改当前的系统环境变量，即系统下的 .profile、.bash_profile 或 .bashrc 文件

NPM 命令

npm init

使用 NPM 安装 Modules 之前，需要先将本地目录 init，会自动创建下述文件及其文件夹：

node_modules：保存各种 Modules 的文件夹
package.json：Modules 管理文件
package-lock.json：Modules 与 Modules 之间依赖关系管理文件

package.json 文件实例

package.json 基本属性

  • name：Modules 名称，其它子模块可以通过 require() 引用该模块
  • author：固定格式：作者 <邮箱>
  • maintainers：贡献者固定格式：作者 <邮箱> (网址)
  • version
  • description
  • license：授权方式
  • homepage：项目网址
  • repository：项目仓库地址

author、version、license 这三个属性，可通过 npm config set 命令对其提前设置，下次 init 时系统会自动为我们填充相关信息

  1. main。
    指定入口文件，当 require() 引用该模块时该文件会被执行

  2. scripts
    例如：上面的 test，当我在终端执行 npm run test 时 gulp 命令会被执行，单纯的只执行 npm run 不加任何参数，系统会列出 scripts 下所有可执行的命令

  3. dependencies
    项目运行所依赖的模块，执行 npm install 时安装的包会注册到此属性下

  4. devDependencies
    项目开发所需要的模块，执行 npm install --save-dev 时安装的包会注册到此属性下

  5. bin
    用来指定各个内部命令对应的可执行文件的位置

上面的例子是，cli 命令对应的可执行文件在当前路径下的 bin 内，NPM 会自动寻找到该文件并在 node_modules/.bin/ 下创建该命令的符号链接（node_modules/.bin/cli）。node_modules/.bin/ 目录会在运行时加入系统的PATH变量，因此在运行 npm 时，就可以不带路径，直接通过命令来调用这些脚本

  1. bugs
    设置使用者提交 bug 的网址及其邮箱
  1. engines
    指定项目运行的 NodeJS 版本范围

package.json 更多属性请查阅 NPM 官方文档

npm search

npm search 在 NPM 公开库搜索 Module

npm install

标准格式命令

Lodash Modules 会安装到本地目录下的 node_modules 内，使用该 Modules 时只需要通过 require(‘Lodash’) 方式引用

安装 package.json 内配置好的模块

npm install 命令其它参数

npm uninstall

npm shrinkwrap

锁定 dependencies 项目运行所依赖的模块，主要目的是为了防止用户随意更改包的依赖造成不必要的麻烦

npm outdated

检测 node_modules 目录内的包是否有可更新版本

指定递归层级

npm update

NPM 指定包更新到最新版本

全部升级到最新版本

npm list

枚列当前安装的所有 Modules 的版本

npm audit

使用此命令检测包是否有安全问题，有的使用 npm audit fix 修复

npm cache

每当你通过 npm 安装一个软件包时，npm 会自动在你系统用户根目录下创建一个副本，并保存到 .npm 目录内，之所以你安装相同的软件包时，速度会更快，那是因为它不需要再访问网络安装了，所以我们需要定期清理

发布自己 Modules

npm init

首先在包目录下初始化，然后根据系统提示依次输入必要的信息，最后生成 package.json。或者执行 npm init -y 直接生成 package.json 跳过提示

npm adduser

注册 npm 资源库账户

npm login

注册过的可使用此命令直接登陆

npm whoami

用于检测当前使用的用户

npm publish

编辑好包后，通过此命令发布到线上
