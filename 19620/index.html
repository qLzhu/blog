更改 Travis 长期 Token

发表于 2021-10-03 更新于 2021-10-04 分类于 hexo 阅读次数： 本文字数： 422 阅读时长 ≈ 1 分钟

本 blog 用的 Github + Travis 方式进行部署的，之前的 Token 有时间限制，由于长时间未管理及其更新，造成了权限丢失。所以，我们此次更换 Travis 项目下的 Token 值。

Travis基础知识

Travis自动化部署

创建 Token

在 Github 上 Settings -> Developer settings -> Personal access tokens -> Generate new token 处创建一个新的 token，重点是 Expiration * 处，要设置成 No expiration，该 token 只开启一个 repo 权限即可。

设置好后，滚动到界面最下方，点击 Generate token 按钮进行创建。注意：创建完的 token 只有第一次可见，之后再访问皆无法再见（只能看见 token 的名称），想查看只能重新再创建，因此要保存好这个值。

部署到Travis

得到长期的 token 后，我们需要回到 Travis 官网，把之前项目下的 token 值更换掉。
